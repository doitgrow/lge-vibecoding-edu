# 신뢰도 평가 프레임워크 (Reliability Scoring)

## 개요

수집된 시장/뉴스 데이터 항목의 신뢰도를 객관적으로 정량화하는 평가 체계입니다.
5개 축, 각 20점 만점, 총 100점 기준으로 평가합니다.

---

## 평가 축 상세

### 축 1: 출처 권위성 (Source Authority) - 20점

출처의 전문성과 공신력을 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| 20 | Tier 1 - 최고 권위 기관/매체 | McKinsey, IHS Markit, 정부 공시, OEM IR |
| 15 | Tier 2 - 전문 매체/경제지 | Automotive News, Bloomberg, TheElec |
| 10 | Tier 3 - 일반 뉴스/보도자료 | 연합뉴스, 기업 뉴스룸 |
| 5 | Tier 4 - 커뮤니티/블로그 | Reddit, 개인 블로그, SNS |
| 0 | 출처 불명/익명 | 출처 표기 없음, "관계자에 따르면" |

**보정 규칙**:
- 1차 출처(원문 보고서) 인용 시: +3점 (최대 20)
- 2차 출처(타 매체 인용) 재인용 시: -2점
- 다수 출처가 동일 팩트 확인 시: +2점

---

### 축 2: 데이터 구체성 (Data Specificity) - 20점

정보의 구체성과 검증 가능성을 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| 20 | 정량 데이터 + 출처 명시 | "2025년 글로벌 EV 시장 $500B (IHS Markit)" |
| 16 | 정량 데이터, 출처 부분 명시 | "시장 규모 30% 성장 (업계 추정)" |
| 12 | 정성적 + 구체적 팩트 | "A사가 B사에 3년 공급계약 체결" |
| 8 | 정성적 + 모호한 정보 | "시장이 빠르게 성장하고 있다" |
| 4 | 의견/전망 위주 | "전문가들은 긍정적으로 본다" |
| 0 | 근거 없는 추측 | "아마도 ~할 것이다" |

**체크 포인트**:
- [ ] 숫자(금액, 비율, 기간)가 포함되어 있는가?
- [ ] 주어(누가)가 명확한가?
- [ ] 시점(언제)이 특정되어 있는가?
- [ ] 제3자가 검증할 수 있는 정보인가?

---

### 축 3: 시의성 (Timeliness) - 20점

정보의 최신성과 현재 유효성을 평가합니다.

| 점수 | 기준 |
|------|------|
| 20 | 오늘~3일 이내 발행 |
| 18 | 4~7일 이내 |
| 15 | 8~14일 이내 |
| 12 | 15~30일 이내 |
| 8 | 1~3개월 이내 |
| 4 | 3~6개월 이내 |
| 0 | 6개월 이상 |

**보정 규칙**:
- 장기 유효 정보(규제 변경 확정, 연간 계획 발표): 발행일 무관 최소 12점 보장
- 빠르게 변하는 정보(주가, 수주 루머): 7일 초과 시 -5점 추가 패널티
- 정기 보고서(분기 실적, 연간 보고): 해당 기간 내이면 15점 보장

---

### 축 4: B2B 관련성 (B2B Relevance) - 20점

자동차 B2B 영업에 대한 직접적 관련도를 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| 20 | 직접 B2B 거래/계약 관련 | "A사, B OEM에 전장부품 5년 수주" |
| 17 | B2B 공급망/파트너십 | "C사-D사 배터리 JV 설립" |
| 14 | 구매/조달 정책 변화 | "현대차, 2차 벤더 직거래 확대 방침" |
| 11 | 산업 구조 변화 (B2B 영향 큼) | "EU, 2030 탄소규제 강화 확정" |
| 8 | 기술 트렌드 (간접 영향) | "SDV 전환으로 소프트웨어 비중 증가" |
| 5 | 일반 산업 동향 | "글로벌 자동차 판매량 전망" |
| 2 | 소비자/B2C 위주 | "신차 출시, 소비자 반응" |
| 0 | 자동차 산업 무관 | |

**판단 기준 질문**:
- 이 정보를 영업 미팅에서 활용할 수 있는가?
- 이 정보가 RFP/RFQ 대응에 도움이 되는가?
- 이 정보로 인해 고객사 접근 전략이 달라지는가?

---

### 축 5: 실행 가능성 (Actionability) - 20점

영업팀이 이 정보를 기반으로 즉시 행동할 수 있는 정도를 평가합니다.

| 점수 | 기준 | 예시 |
|------|------|------|
| 20 | 즉시 실행 가능한 액션 도출 | "다음 주 RFP 마감, 기술 사양 공개됨" |
| 16 | 1~2주 내 액션 가능 | "OEM 구매팀 인사이동, 새 담당자 접촉 필요" |
| 12 | 1개월 내 전략 반영 가능 | "경쟁사 신기술 발표, 제안서 차별화 포인트 추가" |
| 8 | 분기 전략에 참고 | "시장 성장률 전망 상향, 투자 확대 근거" |
| 4 | 장기 방향성 참고 | "2030년 기술 로드맵, 중장기 준비" |
| 0 | 액션 불가 (정보 관찰만 가능) | "업계 일반론, 구체적 행동 불가" |

**실행 가능성 증폭 요소** (있으면 +2~3점):
- 구체적 담당자/기업명이 언급됨
- 데드라인/일정이 특정됨
- 금액/물량 규모가 명시됨
- 경쟁사 동향과 비교 가능

---

## 종합 점수 산출

```
총점 = 출처 권위성 + 데이터 구체성 + 시의성 + B2B 관련성 + 실행 가능성
범위: 0 ~ 100
```

### 등급 분류

| 등급 | 점수 범위 | 의미 | 채택 권장 |
|------|----------|------|----------|
| A+ | 90~100 | 최고 품질, 즉시 활용 | 반드시 포함 |
| A | 80~89 | 높은 품질 | 우선 포함 |
| B+ | 70~79 | 양호 | 포함 권장 |
| B | 60~69 | 보통 | 상위 대체재 없으면 포함 |
| C | 50~59 | 미흡 | 보완 필요 시에만 포함 |
| D | 40~49 | 부족 | 원칙적 제외 |
| F | 0~39 | 불합격 | 제외 |

---

## 평가 실행 템플릿

각 항목을 아래 형식으로 평가합니다:

```markdown
### 항목: {제목}

| 축 | 점수 | 근거 |
|----|------|------|
| 출처 권위성 | XX/20 | {1줄 근거} |
| 데이터 구체성 | XX/20 | {1줄 근거} |
| 시의성 | XX/20 | {1줄 근거} |
| B2B 관련성 | XX/20 | {1줄 근거} |
| 실행 가능성 | XX/20 | {1줄 근거} |
| **총점** | **XX/100** | **등급: {A+~F}** |
```

---

## 동점 처리 규칙

총점 동점 시 우선순위:
1. B2B 관련성 점수가 높은 항목
2. 실행 가능성 점수가 높은 항목
3. 시의성 점수가 높은 항목
4. 그래도 동점이면 양쪽 모두 포함 (10개 초과 허용)

---

## 제약 사항

- 평가자의 주관을 최소화하기 위해 반드시 점수표 기준 준수
- "대략 이 정도" 식 감점/가점 금지 → 점수표의 정확한 기준 적용
- 보정 규칙은 명시된 조건에서만 적용 (임의 보정 금지)
- 평가 근거는 1줄이라도 반드시 기록 (점수만 기재 금지)
